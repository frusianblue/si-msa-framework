## 5. 빌드 & 실행
```bash
# 전체 빌드(컴파일+테스트+JaCoCo+Spotless 검사)
./gradlew build

# 코드 포맷 (최초 1회 전체 정렬 → 단독 커밋 권장)
./gradlew spotlessApply       # 자동 정렬
./gradlew spotlessCheck       # 검사만(CI 게이트)

# 품질/보안
./gradlew dependencyCheckAggregate     # OWASP CVE
./gradlew sonar -Dsonar.host.url=... -Dsonar.token=...

# Flyway (CI 마이그레이션 검증)
./gradlew :services:user-service:flywayValidate -Pflyway.url=... -Pflyway.user=... -Pflyway.password=...

# 로컬 실행 (H2, 실제 로그인 + datasource-proxy SQL 로깅)
./gradlew :services:user-service:bootRun --args='--spring.profiles.active=local'
./gradlew :services:admin-service:bootRun --args='--spring.profiles.active=local'

# 개발 우회 모드 (dev-auth: 토큰 없이 통과 + 가짜 사용자)
./gradlew :services:user-service:bootRun --args='--spring.profiles.active=local,dev'
```
- 시드 계정: `admin / admin123` (ADMIN+USER), `hong / hong123` (USER)
- JWT 공유 시크릿(데모): 서비스 간 동일. user-service 발급 → 타 서비스 검증
- 프로파일: `local`(H2+실로그인), `local,dev`(인증 우회), 기본(PostgreSQL 운영)

## 6. 핵심 설계 결정 (되돌리지 말 것 — 이유 포함)
- **프로젝트별 선택 2단계**: ① 옵션 Gradle 모듈 포함/제외 ② `framework.*.enabled` 프로퍼티 토글.
- **라이브러리 빈 등록은 `@AutoConfiguration` + `.imports`** (서비스 component-scan 범위 밖). 컨트롤러도 `@Bean` 등록.
- **프레임워크는 DB 드라이버를 품지 않음** (재사용성). 각 서비스가 `org.postgresql:postgresql` 선언.
- **TokenStore**: 기본 memory → jdbc fallback → redis(선택). 인터페이스만 공통.
- **인증 추상화**: 프로젝트는 `Authenticator` 하나만 구현. 로그인/refresh/logout·토큰발급은 공통.
- **dev-auth**: prod 프로파일에서 켜지면 **부팅 실패**(DevAuthSafetyGuard). 기본 false.
- **S3는 AWS SDK v2 직접 사용**: Spring Cloud AWS 4.0이 Jackson2 요구 → 충돌 회피.
- **라이브러리 모듈의 spring-security 는 `compileOnly`**: `@PreAuthorize`(commoncode/file) + `AuditLogAspect`(core)의 SecurityContextHolder. 강결합 회피.
- **DDL ID는 `BIGINT GENERATED BY DEFAULT AS IDENTITY`** (PostgreSQL + H2 PG모드 공용).
- **Flyway 운영 마이그레이션(`db/migration`)과 dev 시드(`db/seed-local`) 분리**.
- **(추가) 버전 카탈로그 단일 소스 + ext 브리지**: `libs.versions.toml` 로 일원화, 루트 `ext`로 기존 모듈 호환.
- **(추가) 품질 게이트**: JaCoCo→Sonar 커버리지, Spotless(Palantir) 포맷 강제.
- **(추가) Flyway Gradle 플러그인**: 앱 기동 마이그레이션과 별개로 CI 에서 `flywayValidate` 배포 게이트.
- **(추가) Dockerfile 런타임 전용**: jar 는 Jenkins 빌드, 컨테이너는 레이어 추출만. 엔트리포인트 `JarLauncher`.

## 7. 주의사항 / 알려진 이슈
- ✅ **gradle-wrapper.jar** — 생성 완료(`gradle wrapper --gradle-version 8.14`).
- ✅ **Boot 4 API 변경 대응 완료**(되돌리지 말 것):
  - Jackson3: `JsonMapperBuilderCustomizer`, `WRITE_DATES_AS_TIMESTAMPS` → `cfg.DateTimeFeature`(기본 false)
  - Spring7: `ContentCachingRequestWrapper(request, MAX_BODY)` (단일인자 제거)
  - Security7: `AuthorizationManager.authorize()` + `Supplier<? extends Authentication>`
  - framework-core: `AuditLogAspect` 위해 `spring-security-core` compileOnly 추가
  - AOP `spring-boot-starter-aspectj` / Gateway `spring-cloud-starter-gateway-server-webflux`
- ⚠️ **Testcontainers** — Boot BOM 미관리. 루트 dependencyManagement 에 `testcontainers-bom:1.20.6` 명시.
- framework-core는 security/dao 의존 없음(compileOnly 제외) → `AccessDeniedException`은 RestAccessDeniedHandler, `DataIntegrityViolationException`은 framework-mybatis advice가 처리.
- 첫 `spotlessApply` 는 전 소스 재정렬되므로 **단독 커밋**으로 분리.

## 9. 빠른 검증 (인계 후 동작 확인)
```bash
./gradlew build      # 전체 통과 확인
# 로그인 → 토큰
RESP=$(curl -s localhost:8080/api/v1/auth/login -H 'Content-Type: application/json' \
  -d '{"loginId":"admin","password":"admin123"}')
ACCESS=$(echo $RESP | sed -E 's/.*"accessToken":"([^"]+)".*/\1/')
# 메뉴 / 공통코드(캐시) / 파일 업로드
curl localhost:8080/api/v1/menus/me            -H "Authorization: Bearer $ACCESS"
curl localhost:8080/api/v1/common-codes/GENDER -H "Authorization: Bearer $ACCESS"
curl -X POST localhost:8080/api/v1/files -H "Authorization: Bearer $ACCESS" -F "file=@./test.pdf"
```
